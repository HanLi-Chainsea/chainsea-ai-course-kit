# 如何跑筆記本(.ipynb)— 卡在 ipykernel 看這頁

## 你遇到的錯誤
```
Running cells with 'Python 3.x' requires the ipykernel package.
... error: externally-managed-environment
```
**原因**:系統 Python 被作業系統「鎖住」(externally-managed),不准 `pip install` ── 所以那個「Install」按鈕和 `pip install ipykernel` 都會失敗。**這不是筆記本的問題,也不能用筆記本裡的一格去裝**(kernel 沒 ipykernel 就起不來,連那格都跑不了)。

## 解法:用 venv(乾淨、安全,不碰系統 Python)

打開**終端機(Terminal)**,在學員包根目錄貼這兩行、按 Enter:

```bash
git pull        # 抓最新(才有 setup.sh)
bash setup.sh   # 建 venv + 裝 ipykernel + 註冊「Python (課程)」kernel(約 30 秒)
```

裝完 → 回筆記本 → 右上角 **Select Kernel** → 選 **「Python (課程)」** → 一路按 ▶。

> 為什麼用 venv 不用 `pip install --break-system-packages`?後者會動到系統 Python,有風險;venv 只在專案裡,安全乾淨。

> 在 Codespaces 用「Open in Codespaces」開的話,這步**已經自動幫你做好**,直接選「Python (課程)」就行。

## 完全不想搞環境?直接跑也行(零安裝)
這些純內建套件,不用 ipykernel、不用 venv:
```bash

cd m12-crew-starter     && python3 run.py     # M12 三角色(需金鑰)
```
